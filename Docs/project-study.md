# Tide 项目认知

更新时间：2026-07-16

## 当前作品

Tide 是严格侧视的 2D 生存/环境操作原型。主角因海难来到外海贫瘠岩礁，借死难船骸留下的原物修补临时住所与逃生帆船，逐步读懂潮、水、风、雨和房屋结构，最终在暴潮夺走归处后驶向灯塔。

当前主题不是“建造永恒基地”，而是：

**把临时的家修到足够温暖，并从中长出离开的能力。**

## 核心循环

1. 观察月相、潮窗、风、天气与屋体状态。
2. 在低潮或安全时段拆船骸、布网、收雨、拉船、短航和打捞。
3. 把同一件实物投入住所或逃生船，获得可见结构变化。
4. 更稳的屋、更可靠的船和更早的预报扩大下一轮行动余量。
5. 暴潮时不能无代价保住全部物资，损失会延续到饮水、取暖、修船或航线。

正反馈是“理解自然 -> 布置和取舍更好 -> 真实结构改善 -> 能走更远/看得更早”，不是金币或抽象升级点。

## 当前第一切片

- 唯一场景：`Tide_StiltHouse_FirstSlice`
- 左侧：有限裸岩岛、海难船骸、雨水池和住所材料交付。
- 中段：多层高脚屋、码头、网与潮前设施。
- 右侧：泊位、甩绳拉船、短航海域、危险与灯塔方向。
- 室内外读取同一时间、天气、潮位、屋体损伤和物资身份。

## 设计约束

- 现实世界逻辑优先。任何操作都要能解释力、材料、位置和后果。
- 所见即所得优先。楼板、梯子、船舷、浪面和碰撞必须同源。
- 玩法不依赖左下角说明、箭头、菱形或发光圈。
- 风与流独立；帆只响应有符号风，潮水不等于固定左向阻力。
- 海面只有一个权威场；局部浪花不允许制造第二水线。
- 资源按 owner 原子切换，原物不复制，历史版本不同时驻留。

## 当前风险

1. 大 Controller 仍承担过多 presenter 编排，需要在行为稳定后继续拆分。
2. 房屋、人物、船的最终投影清晰度、比例和接触动画尚未视觉签字。
3. 岩礁岛目前是程序化首版，正式分层资源待交付。
4. 暴潮抢救需要真人 Play 调参，避免四件物资都能轻松保住。
5. macOS Metal、Atlas、加载驻留和完整玩家构建尚未验证。
