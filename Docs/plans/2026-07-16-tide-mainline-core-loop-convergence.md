# Tide 主线与核心循环收敛实施计划

**目标：** 将唯一开发环境收敛到 `D:/UnityEditor/Projects/Tide`，删除历史资源膨胀，并把“贫瘠岩礁岛 -> 海难船骸启动 -> 守屋/修船 -> 风浪短航 -> 暴潮取舍”做成可玩的连续循环。

**架构：** 场景仍使用 `Tide_StiltHouse_FirstSlice`，但世界横向分成船骸岩礁、归处、高脚屋泊位、短航海域四个连续区段。潮汐、风、雨、船体、物资各自保持独立状态模型，场景控制器只编排状态和表现；所有可见物、碰撞和交互共享同一组世界锚点。

**技术栈：** Unity 2022.3 LTS、URP 2D、C#、Git LFS、PowerShell/Bash 静态门。

## P0 工程收敛

1. 迁移 C 盘工作树全部项目源文件到 D 盘主仓库，并核对目录差异和根文件哈希。
2. 删除 C 盘工作树，只保留 `main` 一个工作树。
3. 以 Scene/Prefab/Resources/Catalog 的 GUID 依赖闭包保留当前运行资源。
4. 只保留 Scene、Prefab、Resources、Catalog 或代码直接引用的运行闭包；尚未接入的候选包留在资源会话工作目录，不提前进入 Git。
5. 删除本地截图、旧原型视觉证据、一次性 Python 生成器与重复设计文档。
6. 二进制运行资源走 Git LFS；Library、Temp、Logs、UserSettings、自动截图和资源 QA 输出不入库。

## P0 世界与叙事起点

1. 在高脚屋左侧新增一屏完整岩礁岛，镜头同时看见左右海岸。
2. 地貌只出现裸露黑色基岩、碎石、盐霜和被同一风向压低的稀疏植被；禁止树、肥沃土和隐藏森林暗示。
3. 最左侧放置同一艘海难大船的船骸。玩家拆取木板、帆布、金属件后，船骸对应 owner 永久消失，材料实体转移到玩家或施工位。
4. 高脚屋材料保留工业编号、铆钉和运输标记，表明它依靠漂来/拆取材料而非岛上木材。
5. 淡水采用“屋顶雨槽 -> 开裂蓄水池”系统：雨天积水、晴天蒸发、暴潮盐水倒灌；盐线记录最近一次污染高度。

## P0 可重复核心玩法

1. **绳索：** 瞄准、甩绳、挂住、收紧、放松、解开；张力由距离、相对速度、浪面和锚点强度共同决定。拉船时人物和船都受力，不能瞬移。
2. **短航：** 船只只做左右速度、纵向浮沉和俯仰越浪。玩家可升降帆、调整帆面、移动压舱、舀水和现场修船。
3. **风：** 开场近静风；之后连续变化。顺逆风只改变帆产生的推力，水流独立存在，不再永久给船一个向左阻力。
4. **暴潮：** 海水进入低层后，松动物资按浮力、系固状态和局部流速移动。玩家时间不足以全救，必须在饮水、船材、取暖、线索之间取舍。
5. **反馈：** 每次成功不是加金币，而是船骸 visibly 少一块、屋体多一个真实固定件、蓄水池变可用、船能多走一屏、瞭望能更早发现风暴。

## 验收门

1. 所见即所得：可见平台、梯子、船舷、绳索、浪面与碰撞/锚点误差不超过角色脚宽的四分之一。
2. 真实时间：昼夜可以压缩；呼吸、走路、甩绳、升帆、浪和船体响应使用现实可读频率，不随游戏日倍率加速。
3. 清晰度一致：人物、房屋、船和近景岩礁在目标 16:9/4:3 视图拥有同档像素密度和过滤策略。
4. 深度合理：后海/船后层/人物/前船舷/前景浪的排序固定；进水物资按浮力与遮挡穿过水面。
5. Playtest：不看文字也能在五分钟内发现船骸可拆、雨水可收、船可拉、帆可升、暴潮会带走未固定物。
