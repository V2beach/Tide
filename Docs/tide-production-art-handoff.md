## 协作边界

资源会话负责：

1. 制作透明分层运行资源、锚点和运行契约。
2. 保证同一物件在完整、拆下、手持、施工位之间身份连续。
3. 输出单一推荐运行档，不把 Source、QA、GIF、UHD 或候选板塞入仓库。
4. 候选包先留在会话工作目录；运行侧确认 owner、锚点和用途后，才把被选中的最小运行闭包放进仓库。
5. 在本文末尾按固定格式回写交付，不直接修改玩法状态机。

运行会话负责：

1. 审核资源是否符合世界逻辑、视角、比例、清晰度和所有权。
2. 原子接入 Catalog、Renderer、排序、碰撞和状态机。
3. 退役旧 owner，保证同一像素和同一物件不会双显。
4. 用 Unity 编译、纯状态探针和用户原始 Game View 验收；自动截图不能代替视觉签字。

## 通用运行合同

每组资源必须提供 `runtime-contract.json`，至少声明：

- `canvas_px`、`ppu`、`pivot`、`world_size_m`
- 世界锚点和语义接触点，单位同时给像素与米
- 每个文件的 owner、排序层建议、互斥 owner
- 动画帧序、帧率范围、循环/一次性规则、锚区逐帧漂移上限
- 完整态与移除态的像素重建关系
- 禁止烘焙的内容：天空、整片海水、UI、提示标记、碰撞、动态光色
- 推荐运行档与预计显存；默认只提交 Balanced

硬门：

- 严格正侧视，不得混入俯视甲板、俯视漩涡或错误透视。
- 透明背景，不得带矩形天空或第二片海水。
- 不得出现菱形、箭头、光圈、描边交互点或无法解释的装饰件。
- 人物、房屋、船在最终投影下使用相近 texel/m；不能靠运行缩放修正动作帧大小。
- 碰撞使用契约锚点与可走面，不按 Alpha 边缘自动猜。
- 原物拆下后，母体对应部位必须永久缺失；不能复制一份相似材料。

## ART-P0 岩礁岛与海难起点

状态：`REQUESTED / PROCEDURAL FALLBACK ACTIVE`

### 画面目标

左侧是一座能同时看见两侧海岸的外海小岩礁，不是大陆边缘：

- 主体为黑灰裸露基岩、节理、碎石和盐霜，土层极薄。
- 不出现树、树桩、落叶、肥沃泥土或可疑森林轮廓。
- 植物仅有贴地草、苔藓、矮灌木，全部被同一主风压向右侧。
- 高脚屋材料带旧工业编号、铆钉、运输标记和盐蚀，说明木材与金属来自外部。
- 屋顶排水沟连续通向开裂蓄水池；池壁盐线是独立 owner，可随盐侵高度变化。
- 最左岸是造成主角到达的同一艘大型海难船骸。它不是重复摆放的小木肋，也不是可无限刷新资源点。

### 固定世界范围

- 玩家可走区：`x=-14.55m..-6.90m`
- 开场脚点：`x=-12.45m`
- 蓄水池中心：`x=-7.35m`
- 住所材料交付点：`x=-5.65m`
- Y 基准必须读取当前岩板/码头的实际可走面，不另画悬浮平台。
- 16:9 与 4:3 都要能读到岛屿有限边界；允许相机跟随，不要求把整个高脚屋和整岛强塞进一屏。

### 必须分层

1. `IslandRockBack`：远侧裸岩轮廓，不拥有可走面前缘。
2. `IslandRockWalkable`：脚下岩板、裂隙和碎石；给出可走折线。
3. `IslandRockFront`：前景遮挡岩与浪打湿边，不拥有海水。
4. `SaltAndCracks`：盐霜、风化裂隙，可独立调 Alpha。
5. `WindPlant_A/B/C`：至少三种非重复低矮植被。
6. `GutterAndPipe`：屋檐到池口连续路径，不能断在空中。
7. `CisternBody`：开裂石/混凝土蓄水池，提供池口、内底、裂缝、溢流口锚点。
8. `CisternWaterSurface`：当前存水高度 owner，必须裁在池内，不能烘焙固定水位。
9. `CisternSaltLine`：历史最高盐痕 owner；退水后保留，不得与当前水面合并。
10. `CisternLeakStream`：从主裂口落到同一裸岩面的细流；粗细读取裂缝和当前水量，密封后关闭。
11. `CisternRivetedPatch`：铆接板的最终 owner；试装前原板在池边，固定后才覆盖裂口。
12. `WreckBase`：大型船骸剩余主体。
13. `WreckHullPlank`、`WreckSailcloth`、`WreckRivetedPlate`：三件可拆原物。

### 船骸状态合同

每件可拆物必须有：

- 完整体中的唯一 owner。
- 拆除后的同画布缺件态，缺口位置与原物完全一致。
- 同一原物的手持/拖拽层，不改变轮廓身份和工业标记。
- 住所施工位与逃生船施工位的放置层；两种用途都不能直接变成数字库存。
- 允许将来细分清理、试装、固定、密封，但首版至少完成 `Attached -> Carried -> StagedAtShelter/Boat`。

### 禁止项

- 不要把 V72 六件残骸横向重复成六艘船。
- 不要画完整森林、泥滩、沙滩度假岛或热带植被。
- 不要给船骸加发光交互点、菱形、箭头、文字标签。
- 不要烘焙人物、海水、浪花循环、蓄水液面或天气光色。
- 不要提交 8K QA 板、GIF、UHD 和创作源；它们留在本地工作目录。

## 当前资源侧未关闭问题

1. 房屋 V34/V35、人物 V20/V41/V42、船 V39/V52/V67 的最终投影清晰度仍需统一。
2. 外梯、室内主梯、阁楼梯必须与可走面锚点一一对应；人物不能走出可见楼板。
3. 船上人物必须保留完整双腿，使用 `后船体 < 人物 < 前船舷 < 前景浪` 的自然遮挡；待机不能抽动、缩放跳变。
4. 侧视漩涡必须与同一海面水线融合，泡沫向中心汇入、水下流线更慢；禁止完整俯视圆环。
5. 海况资源只能提供透明浪脊、泡沫和天气层；唯一运行海况由 `TideAuthoritativeOceanModel` 合成连续海体与同一可见浪事件。资源不得烘焙第二水线、船体浮沉、拍击水花位置或独立随机浪力。
