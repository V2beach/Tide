# Tide 核心机制合同

更新时间：2026-07-16

## 一句话循环

**读风、潮、雨和结构 -> 在有限窗口内布置、拆取、拉船或短航 -> 把同一件实物投入住所或逃生船 -> 获得可见改善 -> 在暴潮中承担没有保住的东西。**

核心不是躲一条上涨水线，也不是把漂物换成金币。玩家逐渐学会让自然替自己搬运、判断什么时候该冒险、什么值得留下，以及何时必须离开。

## 六个核心动词

1. **观察**：从月亮、潮痕、风向器、云、浪、雨槽和房屋异响判断下一段环境。
2. **布置**：把网、绳、桶、木桩、压舱物和待修部件放到真实位置。
3. **拆取**：从同一船骸拆下木板、帆布或金属；母体永久少掉对应原物。
4. **操船**：甩绳拉船、升降帆、左右加速、移动压舱、舀水和现场修补。
5. **固定**：把收获或船骸部件搬到住所/船施工位；暴潮时把重要物件绑到高处。
6. **取舍**：时间、承载和天气不足以完成全部目标，未固定的实物会被带走。

## 正反馈

- 雨槽和蓄水池修好，下一段坏天气前拥有更多可饮水。
- 屋柱、屋顶、灶台和取暖改善，暴潮中多争取真实操作时间。
- 船壳、帆、舱底和舵逐步修复，能走更远并承受更复杂海况。
- 潮尺、海图、瞭望设施和灯塔线索改善，预报更早、更窄，但不直接给唯一答案。
- 玩家对同一自然模型理解更深，收益来自判断，不来自角色数值碾压。

## 单一自然世界

- `TideOceanFieldModel` 唯一提供水位、浪面、坡度、实际米制流速和扰动。
- 月相改变天文潮差，也必须改变实际潮流；高低潮附近流速连续过零。
- 风和水流独立。帆只响应有符号风；开局可近静风，A/D 仍能有限加速。
- 宏观昼夜和潮周期可以压缩；人物动作、绳张力、浪头破碎和船体响应按现实秒运行。
- 局部泡沫、浪冠、漩涡和船底水花不能制造第二水位或第二碰撞面。

## 实物身份

每件重要物必须有稳定身份：

`来源 -> 世界漂移/附着 -> 玩家携带/拖拽 -> 施工位暂放 -> 固定/冲失`

- 同一阶段只允许一个 owner 显示。
- 拆下原物后，来源处必须留下对应缺口。
- 网、手、海水、泡沫和提示不得烘焙进实物 Sprite。
- 进入施工位不等于数字入库；至少要看见实物放在哪里、准备修什么。
- 已经固定在归处或船上的物件不能因玩家死亡隔空消失。

## 风浪短航

- 只做水平速度、浮沉和俯仰，不做自由 2D 海图驾驶。
- A/D 提供有限加速度；松手后继续受风、流、阻力和船体损伤影响。
- W/S 连续升降帆；无风时升帆几乎没有收益，逆风方向不能伪装成固定左向阻力。
- Q/E 前后移动已有压舱物，用于抵消当前浪坡纵倾；不能凭空增加压舱重量。
- 船舱进水影响操纵与风险，舀水和修补占用玩家注意力。

## 暴潮取舍

暴潮不是常规每日水位。只有异常天文潮、风暴增水、浪爬高和真实破口共同成立时，低层才进水。

- 物件按浮力、系固、局部水深和潮流分别移动。
- 玩家必须到实物旁持续固定，不得远程按键全部收走。
- 饮水、船材、柴木、海图对应不同后果；熟练玩家也不应无代价保住全部。
- 修得更好的屋不会变成永不损坏，只会在终局争取更长转移时间。

## 所见即所得

- 可见楼板、梯子、岩面、船舷、绳端和水线就是可走/接触边界。
- 锚点误差目标不超过角色脚宽的四分之一。
- 人物各状态保持同一世界身高与脚点，禁止走路放大、待机缩小或上船裁腿。
- 交互从缺口、连接、姿势和因果反馈读出；调试文字可隐藏，正式玩法不依赖箭头、菱形、发光圈或说明栏。

## 新机制进入门

新机制只有同时满足以下四项才进入第一切片：

1. 使用现有风、潮、雨、结构或实物身份，而不是另起一套小游戏。
2. 改变玩家的时机、位置或取舍，而不只是加一个按钮。
3. 能用严格侧视和当前制作规模清楚表达。
4. 不靠文字说明才能理解，也不让美术承担无法验证的物理职责。
